# google-workspace MCP duplicate removed

## Decision

Removed the `"google-workspace"` entry from `/Users/mundiprinceps/Mundi Princeps/.mcp.json` (Mac) and `/home/mundi/Mundi Princeps/.mcp.json` (Achilles). The globally registered `gws` MCP (in `~/.claude.json` / `/home/mundi/.claude.json`) already provides the same tool surface — both entries pointed to the exact same script `scripts/mcp/google-workspace.sh`.

Result: `.mcp.json` now has 6 servers instead of 7 (`clickup`, `context7`, `github`, `n8n`, `playwright`, `supabase`).

## Rationale

`google-workspace` and `gws` were literal duplicates — same stdio command, same args, same script. Having both registered meant two MCP server processes started per session, exposing the same tool schemas under different namespaces (`mcp__google-workspace__*` and `mcp__gws__*`). Pure overhead.

Per user rule: "keep all tools available, just don't disrupt workflows." Dedup is zero-capability-loss — `gws` global registration survives, so every Sheet/Doc/Drive tool still reachable. Only the dup is gone.

## How to reverse

If for any reason a project-local google-workspace registration is needed (e.g., a tool explicitly requires the `mcp__google-workspace__*` namespace), re-add:

```bash
# Mac
jq '.mcpServers["google-workspace"] = {
  "type": "stdio",
  "command": "/bin/bash",
  "args": ["/Users/mundiprinceps/Mundi Princeps/scripts/mcp/google-workspace.sh"],
  "env": {}
}' "/Users/mundiprinceps/Mundi Princeps/.mcp.json" > /tmp/m.json && mv /tmp/m.json "/Users/mundiprinceps/Mundi Princeps/.mcp.json"

# Achilles — same jq with /home/mundi paths
```

## Verification

```bash
jq '.mcpServers | has("google-workspace")' "/Users/mundiprinceps/Mundi Princeps/.mcp.json"
# → false
ssh achilles-mundi 'jq ".mcpServers | has(\"google-workspace\")" "/home/mundi/Mundi Princeps/.mcp.json"'

jq '.mcpServers | has("gws")' ~/.claude.json
# → true (global, unaffected)
```
